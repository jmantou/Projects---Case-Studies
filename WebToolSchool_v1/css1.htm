Introduction to Cascade Style Sheets

Basic Setup of a Style Sheet

The secret to CSS is the Style Sheet, a number of rules for describing how various elements of a web page will diplay. Style Sheets decribe presentation rules for XHTML & HTML5 page elements

Styles are simply a list of selectors (places in the page that we can modify). Each selector can have a set of style rules and each rule describes some attribute of the selector.

Setting a Style Sheet

  • Begin with the Style tags. The type of style you will be working can be embedded in the head of the page or in a separate file with the extension(.css).
  • Include the style tag in the header area. The style type is always "text/css" but you can omit that in HTML5.
  • Define an element. Use the elemen name to begin the definition of a particular element's style. Later we'll learn how to define other things and objects in the page for styling like classes, ids etc.
  • Use braces ({ }) to enclose the style rules. Each style's rule are enclosed in a set of braces, in a style similar to many programming languages. It is also traditional programming technique to indent inside the braces.
  • Give the rule a name. For example font or background color. There are many CSS rules (very often called attributes) and some IDE's can help automate this procedure. A colon (:) character must always follow the atrribute name.
  • Enter the Attribute's value. Different rules take different values. The attribute value must end with a semicolon as in a programming language's statements.

Example

For example if we want all of our paragraphs to have Arial fonts, white text color and a blue bacground we can add the following style:

p {
color:white;
font-family: Arial, Helvetica;
background-color: blue;
}
