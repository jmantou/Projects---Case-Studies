Introduction to XTML & HTML5

Transition from XHTML to HTML5

For everybody that is used in writing Html markup using the strict syntax rules of XHTML, like lowercase elements, quoting attributes and closing empty elements, no problem!

You can make the jump to HTML5 and bring your XHTML habits as far as coding markup is concerned. This particular HTML5 educational website is coded with this in mind and it's 100% valid. This page combines the benefits of the new HTML5 version with the strictness of syntax of the older XTML 1 version.

The 'Lang' Attribute

It is very good idea to declare the language with the 'lang' attribute. See below what the W3C has to say about this:

'Specifying the language of content is useful for a wide number of applications from linguistically - sensitive searching to applying language specific display properties. In some cases the potential applications for language information are still waiting to catch up, whereas in others, such as detection of language by voice browsers, it is a necessity today. Adding markup for language information to content is something that can and should be done today. Without it, it will not be possible to take advantage of any future developments.'

Character Encoding

There is no need to know the Character Encoding in detail, but basically, it defines the set of valid human readable characters that can be used in the document you are building. It is safest to stick to the UTF-8 character set, which is a universal set that allows all characters from almost all the languages in the world.

No More Type Attribute in HTML5

In HTML5 we no longer need to include a type attribute when we declare Javascript script's or CSS. That is because there aren't any other scripting and styling languages (except CSS and Javascript), that you maybe want to use and therefore differentiate between (such as VBScript that was used sometimes in the past).
